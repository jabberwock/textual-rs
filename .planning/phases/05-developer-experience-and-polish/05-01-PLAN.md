IMPORTANT IMPLEMENTATION NOTE: The research identified that `#[derive(Widget)]` operates on structs but cannot see impl blocks, while `#[on]` and `#[keybinding]` live on methods in impl blocks. The recommended approach from research (Open Question 2, final answer): use TWO macros:
- `#[derive(Widget)]` on the struct: generates widget_type_name, can_focus, on_mount, on_unmount
- `#[widget_impl]` on the `impl Widget for MyStruct {}` block: generates on_event, key_bindings, on_action from method annotations

Since Rust forbids two `impl Widget for X` blocks, the derive CANNOT generate `impl Widget`. Instead, derive should generate a HELPER trait impl or free functions that widget_impl references. The cleanest approach:

**#[derive(Widget)]** generates NO `impl Widget` block. Instead it generates:
```rust
impl #name {
    fn __widget_type_name() -> &'static str { #name_str }
    fn __can_focus() -> bool { #focusable }
    fn __on_mount(&self, id: ::textual_rs::widget::WidgetId) { self.own_id.set(Some(id)); }
    fn __on_unmount(&self, _id: ::textual_rs::widget::WidgetId) { self.own_id.set(None); }
}
```

**#[widget_impl]** applied to `impl Widget for MyStruct { fn render(...) {...} }` injects:
- `fn widget_type_name(&self) -> &'static str { Self::__widget_type_name() }` (if not manually provided)
- `fn can_focus(&self) -> bool { Self::__can_focus() }` (if not manually provided)
- `fn on_mount(&self, id: WidgetId) { self.__on_mount(id); }` (if not manually provided)
- `fn on_unmount(&self, id: WidgetId) { self.__on_unmount(id); }` (if not manually provided)
- `fn on_event(...)` from #[on] annotations (if not manually provided)
- `fn key_bindings(...)` from #[keybinding] annotations (if not manually provided)
- `fn on_action(...)` from #[keybinding] annotations (if not manually provided)

This way the user writes:
```rust

struct MyButton {
    own_id: Cell<Option<WidgetId>>,

impl Widget for MyButton {
    fn render(&self, ctx: &AppContext, area: Rect, buf: &mut Buffer) {
        // custom render
    }

    #[on(SomeMessage)]
    fn handle_some_message(&self, msg: &SomeMessage, ctx: &AppContext) {
        // handle it
    }

    #[keybinding("enter", "press")]
    fn handle_press(&self, ctx: &AppContext) {
        // handle press
    }
}
```
