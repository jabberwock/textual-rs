```rust
pub struct RichLog {
    pub lines: Reactive<Vec<ratatui::text::Line<'static>>>,
    pub scroll_offset: Reactive<usize>,
    auto_scroll: Cell<bool>,
    pub viewport_height: Cell<u16>,
    own_id: Cell<Option<WidgetId>>,
    pub max_lines: Option<usize>,
}
```

2. **Constructor:** `RichLog::new()` — defaults auto_scroll=true, max_lines=None. Add `RichLog::with_max_lines(n: usize)` that sets max_lines=Some(n).

3. **write_line(&self, line: ratatui::text::Line<'static>):**
   - If max_lines is Some(max) and lines.len() >= max, evict head: `lines.drain(0..1)` and if scroll_offset > 0, decrement by 1.
   - Push the new line.
   - Auto-scroll logic: identical to Log::push_line but on the Reactive<Vec<Line>>.

4. **clear(&self):** Reset lines to empty, scroll_offset to 0, auto_scroll to true.

5. **Widget impl:**
   - `widget_type_name` -> "RichLog"
   - `can_focus` -> true
   - `default_css` -> `"RichLog { min-height: 3; flex-grow: 1; }"`
   - `on_mount` / `on_unmount` -> save/clear own_id (identical to Log)
   - `key_bindings` -> Same 4 bindings as Log (Up=scroll_up, Down=scroll_down, Home=scroll_top, End=scroll_bottom) plus PageUp/PageDown
   - `on_action` -> Same scroll logic as Log
   - `render` -> Like Log but use `buf.set_line(area.x, y, &line, text_width)` instead of `buf.set_string`. Reserve last column for scrollbar (text_width = area.width - 1 when area.width > 1). Draw vertical_scrollbar when content overflows.

6. **Add to mod.rs:** `pub mod rich_log;` after the `pub mod radio;` line.

7. **Add to lib.rs:** `pub use widget::rich_log::RichLog;` in the re-export block after the existing `pub use widget::radio::...` line.
