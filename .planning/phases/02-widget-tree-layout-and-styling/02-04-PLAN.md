    1. **App struct fields:**
       ```rust
       pub struct App {
           ctx: AppContext,
           bridge: TaffyBridge,
           stylesheets: Vec<Stylesheet>,
           hit_map: Option<MouseHitMap>,
           root_screen_factory: Option<Box<dyn FnOnce() -> Box<dyn Widget>>>,
       }
       ```
       The `root_screen_factory` is a closure that creates the root screen widget. Users call `App::new(|| Box::new(MyScreen))` to provide it.

    2. **App::new() signature change:**
       ```rust
       pub fn new<F>(screen_factory: F) -> Self
       where F: FnOnce() -> Box<dyn Widget> + 'static
       ```
       Store the factory. Initialize AppContext::new() and TaffyBridge::new(). Stylesheets start empty.

    3. **App::with_css(mut self, css: &str) -> Self** builder method:
       Parse the CSS string via `Stylesheet::parse()`, push to `self.stylesheets`, log any parse errors via eprintln. Return self for chaining.

    4. **App::run(&mut self) -> Result<()>** (change from &self to &mut self):
       - Call init_panic_hook()
       - Create Tokio runtime and LocalSet as before
       - Call self.run_async()

    5. **App::run_async(&mut self)** integration:
       - Create TerminalGuard, Terminal as before
       - Take the root_screen_factory and call it to get the root screen widget
       - Call `widget::tree::push_screen(root_screen, &mut self.ctx)` to mount
       - Call `self.full_render_pass(&mut terminal)` for initial render
       - In the event loop:
         - `AppEvent::Key(Tab)`: call `widget::tree::advance_focus(&mut self.ctx)`, then re-render
         - `AppEvent::Key(Shift+Tab)`: call `widget::tree::advance_focus_backward(&mut self.ctx)`, then re-render
         - `AppEvent::Key(q)` / `AppEvent::Key(Ctrl+C)`: break
         - `AppEvent::Resize(c, r)`: call `self.full_render_pass(&mut terminal)`
         - Other keys: no-op for Phase 2 (event dispatch comes in Phase 3)

    6. **App::full_render_pass<B: Backend>(&mut self, terminal: &mut Terminal<B>)** method:
       This is the core integration — the sequence that makes widgets appear on screen:
       ```
       a. Get active screen id: self.ctx.screen_stack.last().copied()
       b. Apply CSS cascade: apply_cascade_to_tree(screen_id, &self.stylesheets, &mut self.ctx)
       c. Sync layout: self.bridge.sync_dirty_subtree(screen_id, &self.ctx)
       d. Compute layout: self.bridge.compute_layout(screen_id, terminal.size()?.width, terminal.size()?.height)
       e. Clear dirty flags: clear_dirty_subtree(screen_id, &mut self.ctx)
       f. Build hit map: collect DFS widget ids, build MouseHitMap from bridge.layout_cache()
       g. Render: terminal.draw(|frame| self.render_widget_tree(screen_id, frame))
       ```

    7. **App::render_widget_tree(&self, screen_id: WidgetId, frame: &mut Frame)** method:
       Walk the active screen subtree in DFS order. For each widget id:
       - Get the Rect from `self.bridge.rect_for(id)`
       - If Rect is Some and has non-zero area:
         - Get widget ref from `self.ctx.arena[id]`
         - Call `widget.render(&self.ctx, rect, frame.buffer_mut())`
       This is the moment where Widget::render() actually writes cells to the terminal buffer.

    8. **render_frame() backward compat:** Keep a simplified `render_frame` that creates a temporary App for Phase 1 test compatibility, OR remove it and update any tests. Since Phase 1 tests used `App::render_frame()` with TestBackend, add a new `App::render_to_test_backend(&mut self, cols: u16, rows: u16) -> Buffer` method that creates a TestBackend terminal, runs full_render_pass, and returns the buffer. This is the Phase 2 test entry point.

    9. **event.rs update:** No changes needed for Phase 2. Tab/Shift+Tab are handled directly in run_async. Keep AppEvent as-is.

    Note: The `full_render_pass` method calls `apply_cascade_to_tree` on every render. This is acceptable for Phase 2 — the cascade is fast for small trees. Phase 3 can optimize to only re-cascade when styles change.

    Note: The borrow pattern for render_widget_tree requires care. `self.ctx.arena[id]` borrows the arena immutably, then `widget.render(&self.ctx, ...)` borrows ctx again immutably — this is fine since render takes `&AppContext`. The `frame.buffer_mut()` is a separate mutable borrow on Frame, not on self.
