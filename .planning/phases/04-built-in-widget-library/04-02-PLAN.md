pub trait Widget: 'static {
    fn render(&self, ctx: &AppContext, area: Rect, buf: &mut Buffer);
    fn compose(&self) -> Vec<Box<dyn Widget>> { vec![] }
    fn on_mount(&self, _id: WidgetId) {}
    fn can_focus(&self) -> bool { false }
    fn widget_type_name(&self) -> &'static str;
    fn default_css() -> &'static str where Self: Sized { "" }
    fn on_event(&self, _event: &dyn std::any::Any, _ctx: &AppContext) -> EventPropagation;
    fn key_bindings(&self) -> &[KeyBinding] { &[] }
    fn on_action(&self, _action: &str, _ctx: &AppContext) {}
}

    **input.rs (WIDGET-03):**

    Struct:
    ```rust
    pub struct Input {
        pub value: Reactive<String>,     // per D-08
        pub placeholder: String,
        pub password: bool,
        cursor_pos: Cell<usize>,         // byte offset into value string
        own_id: Cell<Option<WidgetId>>,
    }
    ```

    Constructor: `Input::new(placeholder: &str) -> Self` with empty value, password=false.

    Messages (per D-07):
    ```rust
    pub mod messages {
        pub struct Changed { pub value: String }
        impl Message for Changed {}
        pub struct Submitted { pub value: String }
        impl Message for Submitted {}
    }
    ```

    `widget_type_name` returns "Input". `can_focus` returns true.

    Key bindings (all with show: false except noted):
    - Left arrow → "cursor_left"
    - Right arrow → "cursor_right"
    - Home → "cursor_home"
    - End → "cursor_end"
    - Ctrl+Left → "word_left"
    - Ctrl+Right → "word_right"
    - Backspace → "delete_back"
    - Delete → "delete_forward"
    - Enter → "submit"

    `on_event` handles `crossterm::event::KeyEvent` directly for character insertion:
    - Match `KeyEvent { code: KeyCode::Char(c), modifiers: KeyModifiers::NONE | KeyModifiers::SHIFT, .. }` — insert char at cursor_pos, advance cursor, post Changed
    - This is needed because key_bindings can't match arbitrary characters

    `on_action` handlers:
    - "cursor_left": decrement cursor_pos (char boundary aware)
    - "cursor_right": increment cursor_pos (char boundary aware)
    - "cursor_home": set cursor_pos to 0
    - "cursor_end": set cursor_pos to value.len()
    - "word_left": scan backward for word boundary (space/punctuation)
    - "word_right": scan forward for word boundary
    - "delete_back": remove char before cursor, decrement cursor_pos, post Changed
    - "delete_forward": remove char at cursor, post Changed
    - "submit": post Submitted { value }

    `render`:
    - If value is empty and not focused, show placeholder in dim style
    - If password, show '*' repeated for value length
    - Show cursor position indicator (reverse video on the character at cursor_pos, or '|' if at end)
    - Use `get_untracked()` for value reads

    `default_css`: `"Input { border: tall; height: 3; }"`

    Add `pub mod input;` to widget/mod.rs and `pub use widget::input::Input;` to lib.rs.

    **Tests in widget_tests.rs** (append to existing file):
    - `input_type_text`: Create Input, pilot.type_text("hello"), verify value is "hello"
    - `input_cursor_movement`: Type "hello", press Left 2 times, verify cursor at position 3
    - `input_backspace`: Type "hello", press Backspace, verify value is "hell"
    - `input_submit_emits_message`: Type "test", press Enter, verify Submitted message posted
    - `input_placeholder_renders`: Create Input with placeholder "Type here", snapshot empty state shows placeholder
    - `input_password_mode`: Create Input with password=true, type "secret", snapshot shows "******"

    **radio.rs (WIDGET-07):**

    RadioButton struct:
    ```rust
    pub struct RadioButton {
        pub checked: Reactive<bool>,
        pub label: String,
        own_id: Cell<Option<WidgetId>>,
    }
    ```

    RadioButton messages:
    ```rust
    pub mod messages {
        pub struct RadioButtonChanged { pub checked: bool }
        impl Message for RadioButtonChanged {}
        pub struct RadioSetChanged { pub index: usize, pub value: String }
        impl Message for RadioSetChanged {}
    }
    ```

    RadioButton Widget impl:
    - `widget_type_name` returns "RadioButton"
    - `can_focus` returns true
    - Key bindings: Space/Enter → "select" (show: true for Space)
    - `on_action("select")`: set checked to true (not toggle — radio buttons only turn on), post `RadioButtonChanged { checked: true }`
    - `render`: display `"(●) label"` when checked (`\u{25cf}`), `"( ) label"` when unchecked — use `get_untracked()`
    - `default_css`: `"RadioButton { height: 1; }"`

    RadioSet struct:
    ```rust
    pub struct RadioSet {
        pub buttons: Vec<RadioButton>,
        pub selected: Reactive<usize>,
        own_id: Cell<Option<WidgetId>>,
    }
    ```

    RadioSet Widget impl:
    - `widget_type_name` returns "RadioSet"
    - `can_focus` returns false (children are focusable)
    - `compose` returns `self.buttons` as `Vec<Box<dyn Widget>>` — each RadioButton is a child
    - `on_event`: downcast to `RadioButtonChanged`. When received, iterate buttons: find which one is checked (the source), set all others to unchecked, update `self.selected`, post `RadioSetChanged { index, value: button.label.clone() }` (per D-07)
    - `default_css`: `"RadioSet { layout: vertical; }"`

    Constructor: `RadioSet::new(labels: Vec<String>) -> Self` — creates RadioButton for each label, first one checked by default.

    Add `pub mod radio;` to widget/mod.rs. Add to lib.rs:
    ```rust
    pub use widget::radio::{RadioButton, RadioSet};
    ```

    - `radio_button_renders_unchecked`: Snapshot of RadioButton with checked=false

    - `radio_set_emits_changed`: Verify RadioSetChanged message is posted with correct index after selection change.
