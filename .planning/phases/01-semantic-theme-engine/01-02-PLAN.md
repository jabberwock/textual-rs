**Strategy**: The simplest integration point is at parse time. The cssparser crate tokenizes `$primary` as a `Token::IDHash("primary")` (since $ followed by ident is not standard CSS, it will likely be parsed as a hash token) OR it may fail to parse. We need to handle the `$` prefix gracefully.

**Approach A (recommended)**: Two-phase resolution. During parsing, detect `$variable` tokens and store them as a new TcssValue variant. During cascade (in apply_declarations or resolve_cascade), resolve the variable against the active theme.

This is cleaner because:
- parse_declaration_block doesn't need a Theme parameter (preserving its current signature for tests)
- Variables resolve at cascade time, so theme changes don't require re-parsing CSS

**cssparser token handling for $variable**: The `$` character is tokenized as `Token::Delim('$')`. After consuming `$`, we need to read the variable name which may contain hyphens. `input.expect_ident()` will read the first ident segment. Then we need to check for `-lighten-N` or `-darken-N` suffixes. Since cssparser will tokenize `primary-lighten-2` as `Token::Ident("primary-lighten-2")` (hyphens are valid in CSS idents), this should work: after `$`, call `expect_ident_cloned()` to get the full name including hyphens. But `lighten-2` may be parsed as `lighten` + `Delim('-')` + `Number(2)`. Test this carefully.

Alternative simpler approach: After seeing `$`, collect all remaining non-whitespace/non-semicolon characters into a string manually. Use `input.parse_until_before(Delimiter::Semicolon, |inner| { ... })` to grab the raw text.

Actually the most reliable: when we see `$`, use a custom parse loop:
```rust
fn try_parse_variable<'i>(input: &mut Parser<'i, '_>) -> Option<String> {
    let state = input.state();
    if let Ok(Token::Delim('$')) = input.next() {
        let mut name = String::new();
        // Read ident
        if let Ok(ident) = input.expect_ident_cloned() {
            name.push_str(&ident);
        } else {
            input.reset(&state);
            return None;
        }
        // Try to read -lighten-N or -darken-N suffix
        let suffix_state = input.state();
        if let Ok(Token::Delim('-')) = input.next() {
            if let Ok(ident2) = input.expect_ident_cloned() {
                // Should be "lighten" or "darken"
                if ident2 == "lighten" || ident2 == "darken" {
                    if let Ok(Token::Delim('-')) = input.next() {
                        if let Ok(Token::Number { int_value: Some(n), .. }) = input.next() {
                            name.push('-');
                            name.push_str(&ident2);
                            name.push('-');
                            name.push_str(&n.to_string());
                            return Some(name);
                        }
                    }
                }
            }
            // No valid suffix, reset to after base ident
            input.reset(&suffix_state);
        } else {
            input.reset(&suffix_state);
        }
        Some(name)
    } else {
        input.reset(&state);
        None
    }
}
```
