# Phase 5 — UI Design Contract

> Visual and interaction contract for the demo apps and command palette widget in Phase 5.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> NOTE: This is a Rust TUI (terminal UI) project using ratatui + TCSS. There is no web
> frontend, no browser DOM, and no shadcn. The template's web-centric fields (px sizes,
> component library, icon library, web font) are adapted to their terminal equivalents:
> terminal cells, ratatui Color values, TCSS properties, and Unicode box-drawing characters.
> "px" values below mean character cells, not CSS pixels.

## Spacing Scale

Terminal spacing is measured in character cells, not CSS pixels. The 8-point grid maps to
terminal cell multiples: 1 cell = 1 character column or row.

| Token | Value | Terminal equivalent | Usage |
|-------|-------|---------------------|-------|
| xs | 1 cell | 1 column/row padding | Inline icon gap, tight label spacing |
| sm | 2 cells | 2 column/row padding | Compact widget internal padding |
| md | 4 cells | 4 column/row padding | Default widget padding (TCSS `padding: 0 2`) |
| lg | 6 cells | 6 column/row gap | Panel/section separation |
| xl | 8 cells | 8 column/row gap | Major layout region gaps |
| 2xl | 12 cells | 12 column/row gap | Demo screen outer padding |
| 3xl | 16 cells | 16 column/row gap | Full-screen content margin |

Exceptions:
- Command palette overlay: fixed width of 60 columns centered, height of min(20, content + 2)
- Header bar: exactly 1 row tall (dock: top; height: 1)
- Footer bar: exactly 1 row tall (dock: bottom; height: 1)
- Tab bar: exactly 1 row tall (height: 1)
- Input/TextArea widgets: minimum 3 rows (1 content + 1 border top + 1 border bottom)

Source: default terminal layout conventions; exceptions from CONTEXT.md D-10, D-11.

## Typography

Terminal typography has no font size or weight selection — the terminal font is monospaced and
fixed by the user's terminal configuration. "Weight" in TUI context means ratatui Modifier flags.
"Size" is not applicable for terminal cells except via spacing/proportion.

| Role | Ratatui Modifier | TCSS equivalent | Usage |
|------|-----------------|-----------------|-------|
| Body | none (normal) | (default) | All standard text, log lines, list items |
| Label | BOLD | `text-style: bold` | Widget titles, column headers, tab labels |
| Heading | BOLD + color: Accent-Green | `color: #00ffa3; text-style: bold` | Screen section headings, demo category titles |
| Display | BOLD + color: Accent-Cyan | `color: #00d4ff; text-style: bold` | App title in Header, command palette title |

Line height: 1.0 (terminal cells are always exactly 1 row tall — no configurable line height).

Source: D-12 (lazeport palette); ratatui Modifier system (default for TUI context).

## Color

All colors are TCSS/ratatui Color values. Hex values are 24-bit true color (requires truecolor
terminal). Terminal fallback approximations listed for 256-color terminals.

| Role | Hex value | ANSI 256 approx | Usage |
|------|-----------|-----------------|-------|
| Dominant (60%) | `#0a0a0f` | Color::Black (ANSI 0) | Main terminal background for all screens |
| Secondary (30%) | `#12121a` | Color::Indexed(233) | Panel backgrounds, sidebar, overlay backgrounds, bordered containers |
| Accent-Green (10% shared) | `#00ffa3` | Color::Indexed(48) | Active selection highlight, focused widget border, primary CTA button |
| Accent-Cyan (10% shared) | `#00d4ff` | Color::Indexed(45) | App title, keybinding hints in Footer, command palette match highlight |
| Muted (supporting) | `#4a4a5a` | Color::Indexed(239) | Inactive/secondary text, placeholder text, unfocused borders |
| Destructive | `#ff4444` | Color::Indexed(196) | Error state text, validation error borders, destructive action copy |

Accent-Green reserved for:
- Focused widget border (`:focus` pseudo-class `border: green #00ffa3`)
- Active tab underline indicator
- Primary CTA button background (Button variant `primary`)
- Selected list item highlight background
- Command palette — currently selected command row

Accent-Cyan reserved for:
- App title text in Header widget
- Keybinding labels in Footer widget
- Command palette title bar text
- Fuzzy match highlight spans within command palette results

Source: CONTEXT.md D-12 ("lazeport.pwn.zone — deep void dark backgrounds, green accent
#00ffa3, cyan highlights #00d4ff, muted secondary text"). Dominant/secondary derived from
lazeport void dark aesthetic.

## TCSS Rules — Demo Stylesheet

The following TCSS rules define the visual contract for `demo.rs` and `irc_demo.rs`. These are
prescriptive defaults for the demo app stylesheets. Tutorial examples use simpler subsets.

```css
/* Base screen */
Screen {
    background: #0a0a0f;
    color: #c8c8d8;
}

/* Panels and containers */
.panel {
    background: #12121a;
    border: solid #4a4a5a;
    padding: 0 1;
}

/* Focus state — all focusable widgets */
*:focus {
    border: solid #00ffa3;
}

/* Header */
Header {
    background: #12121a;
    color: #00d4ff;
    text-style: bold;
    height: 1;
    dock: top;
}

/* Footer */
Footer {
    background: #12121a;
    color: #00d4ff;
    height: 1;
    dock: bottom;
}

/* Tabs */
Tabs {
    background: #0a0a0f;
    height: 1;
}

/* Active tab */
Tab.-active {
    color: #00ffa3;
    text-style: bold;
}

/* Primary button */
Button.primary {
    background: #00ffa3;
    color: #0a0a0f;
    text-style: bold;
}

/* Muted / secondary text */
.muted {
    color: #4a4a5a;
}

/* Error / validation */
.error {
    color: #ff4444;
    border: solid #ff4444;
}

/* Command palette overlay */
CommandPalette {
    background: #12121a;
    border: rounded #00d4ff;
    width: 60;
    padding: 1 2;
}
```

Source: CONTEXT.md D-12 (palette); RESEARCH.md architecture patterns (TCSS property support
confirmed in CSS-06).

## Interaction Contracts

### Command Palette (DX-04)

| Interaction | Behavior |
|-------------|----------|
| Trigger | `Ctrl+P` — opens CommandPalette as screen overlay via `push_screen_deferred` |
| Dismiss | `Esc` — pops the overlay screen |
| Search | Type to filter commands with fuzzy matching (strsim Jaro-Winkler) |
| Navigate | `Up`/`Down` arrow keys move selection; `Enter` executes and dismisses |
| Display columns | Command name (left-aligned), source widget type (muted), keybinding (cyan, right-aligned) |
| Empty results | Show "No commands match '{query}'" in muted color |
| Match highlight | Matched characters rendered in Accent-Cyan (`#00d4ff`) |

### Demo App (demo.rs) Widget Showcase

| Element | Behavior |
|---------|----------|
| Tab navigation | `Tabs`/`TabbedContent` with 4 tabs: Inputs, Display, Layout, Interactive |
| Tab switch | Mouse click or `Left`/`Right` arrow keys when Tabs widget focused |
| Widget focus | `Tab`/`Shift+Tab` cycles through interactive widgets within each tab |
| Quit | `q` or `Ctrl+C` exits the application |
| Demo header | Shows "textual-rs demo" in Accent-Cyan, subtitle in muted |

### IRC Demo (irc_demo.rs)

| Element | Behavior |
|---------|----------|
| Layout | Sidebar (channel list, 20 cols wide) + main chat area + input at bottom |
| Channel switch | `Up`/`Down` in sidebar ListView; `Enter` to join/switch channel |
| Message input | `Input` widget at bottom, `Enter` to send |
| Chat display | `Log` widget with auto-scroll enabled |
| Quit | `q` exits |

Source: CONTEXT.md D-08, D-10, D-11; RESEARCH.md architecture patterns.

## Copywriting Contract

(minimum 80 columns assumed).

| Element | Copy | Location |

| Palette trigger hint | `ctrl+p Command Palette` | Footer key binding label |

user actions in the demo apps or API. The derive macro and worker cancellation are
developer-facing and have no confirmation UI.
