# Phase 5 Plan 1: Proc-Macro Crate (Widget Derive + widget_impl) Summary

Implemented proc-macro crate `textual-rs-macros` providing `#[derive(Widget)]` and `#[widget_impl]` — eliminates ~20 lines of Widget boilerplate per widget via inherent helper methods and annotation-driven method injection.

## Objective Achieved

The `textual-rs-macros` crate is live and tested. Users can now write:

```rust
#[derive(textual_rs::Widget)]
#[focusable]
struct MyButton {
    own_id: Cell<Option<WidgetId>>,
    label: String,
}

#[textual_rs::widget_impl]
impl Widget for MyButton {
    fn render(&self, ctx: &AppContext, area: Rect, buf: &mut Buffer) {
        // only the unique logic
    }

    #[on(SomeMessage)]
    fn handle_some_message(&self, msg: &SomeMessage, ctx: &AppContext) { ... }

    #[keybinding("enter", "press")]
    fn handle_press(&self, ctx: &AppContext) { ... }
}
```

And get a complete Widget impl with widget_type_name, can_focus, on_mount, on_unmount, on_event, key_bindings, and on_action — all generated.

## Tasks Completed

### Task 1: Create textual-rs-macros crate and implement #[derive(Widget)]
- Created `crates/textual-rs-macros/` as proc-macro workspace member
- `#[derive(Widget)]` generates inherent `impl StructName` with `__widget_type_name()`, `__can_focus()`, `__on_mount()`, `__on_unmount()` helper methods
- `#[focusable]` attribute on struct makes `__can_focus()` return `true`

- Commit: `5cdead0`

### Task 2: Implement #[widget_impl] attribute macro with #[on] and #[keybinding] support, plus tests
- `#[widget_impl]` on `impl Widget for T` block:
  - Injects `widget_type_name`, `can_focus`, `on_mount`, `on_unmount` delegating to `__` helpers (unless manually provided)
  - Extracts `#[on(MsgType)]` annotated methods out of the Widget impl into a separate inherent impl, generates `on_event` dispatch chain
  - Extracts `#[keybinding("key", "action")]` methods into inherent impl, generates `key_bindings()` and `on_action()` match dispatch
- 7 integration tests in `crates/textual-rs/tests/macro_tests.rs` — all pass
- Commit: `1104f8a`

## Decisions Made

1. **Handler methods extracted to inherent impl**: Methods with `#[on(T)]` and `#[keybinding]` cannot remain in `impl Widget for T` because they are not Widget trait methods. The macro extracts them to a separate `impl StructName` block that Rust appends after the Widget trait impl.

2. **OnceLock for key_bindings**: The `key_bindings()` method generated by `#[keybinding]` uses `OnceLock<Vec<KeyBinding>>` rather than a `static &[KeyBinding]` slice, because crossterm `KeyCode`/`KeyModifiers` values cannot be evaluated at compile-time in a `const` context (no `const` constructors available).

3. **Same-name re-export is safe**: `pub use textual_rs_macros::Widget` does not conflict with `pub use widget::Widget` (the trait) — Rust separates derive macro names from type/trait names in its namespace.
