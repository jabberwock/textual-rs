Create the widget tree foundation: SlotMap arena, Widget trait, AppContext, compose/mount/unmount lifecycle, screen stack, focus management, and shared CSS type definitions.

CSS type definitions (ComputedStyle, property enums) are included here because AppContext's SecondaryMaps reference them.

Output: Compiling `widget` and `css::types` modules, with unit tests proving arena CRUD, compose traversal, screen stack push/pop, and DOM-order focus traversal.

    1. `pub fn mount_widget(widget: Box<dyn Widget>, parent: Option<WidgetId>, ctx: &mut AppContext) -> WidgetId`:
       - Insert widget into `ctx.arena`, get `id`
       - Insert into all SecondaryMaps: `children[id] = vec![]`, `parent[id] = parent`, `computed_styles[id] = ComputedStyle::default()`, `inline_styles[id] = vec![]`, `dirty[id] = true`, `pseudo_classes[id] = PseudoClassSet::default()`
       - If `parent` is Some, push `id` to `ctx.children[parent]`
       - Remove widget from arena temporarily (`let w = ctx.arena.remove(id).unwrap()`), call `w.on_mount(ctx, id)`, re-insert at same id — wait, DenseSlotMap doesn't support insert-at-id. Use a different approach: call `on_mount` AFTER insertion by getting widget ref. But on_mount takes `&mut AppContext` which conflicts with arena borrow.
       - CORRECT APPROACH (per RESEARCH Pitfall 1): Remove the widget from arena, call on_mount, re-insert. BUT DenseSlotMap doesn't have insert-at-key. Instead, use this pattern: collect on_mount calls and defer them. Or simpler: since `on_mount` default is a no-op and Phase 2 widgets don't override it yet, just skip calling on_mount for now and add a TODO comment. The mount/unmount lifecycle will be fully wired in Phase 3 when the message pump exists.
       - Actually, the cleanest approach: store `on_mount` calls in a pending queue on AppContext. Add field `pub pending_mounts: Vec<WidgetId>` to AppContext. After mounting, push id to pending_mounts. A separate `pub fn flush_pending_mounts(ctx: &mut AppContext)` drains the queue and calls on_mount for each by temporarily removing from arena.
       - Return `id`

    2. `pub fn unmount_widget(id: WidgetId, ctx: &mut AppContext)`:
       - Collect all descendant ids via DFS (children first, then self) into a `Vec<WidgetId>` — bottom-up order
       - For each descendant (bottom-up): remove from arena (calling on_unmount via temporary removal same pattern), remove all SecondaryMap entries
       - Remove `id` from parent's children list if parent exists
       - If `ctx.focused_widget == Some(id)` or any descendant, set `ctx.focused_widget = None`
       - Log warning (eprintln!) if id not found in arena per UI-SPEC: "WidgetId {id:?} not found in arena -- unmount is a no-op"

    3. `pub fn compose_children(parent_id: WidgetId, ctx: &mut AppContext)`:
       - Get the widget's compose() output: `let w = &ctx.arena[parent_id]; let children = w.compose();`
       - For each child in children: call `mount_widget(child, Some(parent_id), ctx)`

    4. `pub fn push_screen(screen: Box<dyn Widget>, ctx: &mut AppContext) -> WidgetId`:
       - `let id = mount_widget(screen, None, ctx);`
       - `ctx.screen_stack.push(id);`
       - `compose_children(id, ctx);`
       - Return id

    5. `pub fn pop_screen(ctx: &mut AppContext) -> Option<WidgetId>`:
       - Pop from screen_stack. If Some(id), call `unmount_widget(id, ctx)`. Return the id.

    6. `pub fn advance_focus(ctx: &mut AppContext)`:
       - Get current screen id from `ctx.screen_stack.last()`. Return if no screen.
       - Collect all widget ids in depth-first order from the screen subtree where `widget.can_focus() == true`
       - If focusable list is empty, return
       - Find current focused index. If None or not in list, focus first. Otherwise focus next (wrapping).
       - Update `ctx.focused_widget`. Update pseudo_classes: remove Focus from old, add Focus to new.

    7. `pub fn advance_focus_backward(ctx: &mut AppContext)`:
       - Same as advance_focus but move to previous (wrapping from first to last).

    8. `pub fn mark_widget_dirty(id: WidgetId, ctx: &mut AppContext)`:
       - Set `ctx.dirty[id] = true`
       - Walk ancestors via `ctx.parent[id]`, setting dirty=true, stopping if already dirty

    9. `pub fn clear_dirty_subtree(root: WidgetId, ctx: &mut AppContext)`:
       - Set `ctx.dirty[root] = false`
       - Recursively clear children

    10. Helper `fn collect_subtree_dfs(root: WidgetId, ctx: &AppContext) -> Vec<WidgetId>`:
        - DFS traversal returning all ids including root

    11. Helper `fn collect_focusable_dfs(root: WidgetId, ctx: &AppContext) -> Vec<WidgetId>`:
        - DFS traversal returning only ids where `ctx.arena[id].can_focus() == true`

Alternative: just use the approach where on_mount takes `&self` instead of `&mut AppContext`. Let me reconsider.)
