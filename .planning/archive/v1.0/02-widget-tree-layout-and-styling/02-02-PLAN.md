<objective>
Integrate the Taffy layout engine with the widget arena: TaffyBridge syncs widget tree structure to Taffy nodes, converts ComputedStyle to Taffy Style, computes layout, and converts f32 geometry to ratatui Rect. Includes dock layout via absolute positioning, dirty-flag-driven incremental relayout, and mouse hit map.

Purpose: This gives every widget a concrete screen position (Rect) computed from its CSS-like styling properties. Without this, widgets have no idea where to render. The TaffyBridge is the bridge between the declarative styling world and the concrete terminal cell grid.

Output: Compiling `layout` module with unit tests proving all layout modes (flex vertical/horizontal, grid, dock, fractional, fixed/percent/auto) produce correct Rect values.
</objective>

       ```rust
           pub fn compute_layout(&mut self, screen_id: WidgetId, cols: u16, rows: u16) {
               let root = self.node_map[&screen_id];
               let space = taffy::geometry::Size {
                   width: taffy::style::AvailableSpace::Definite(cols as f32),
                   height: taffy::style::AvailableSpace::Definite(rows as f32),
               };
               self.tree.compute_layout(root, space).unwrap();

               // Cache Rects: floor() for position, round() for size
               for (&wid, &nid) in &self.node_map {

                           x: layout.location.x.floor() as u16,
                           y: layout.location.y.floor() as u16,
                           width: layout.size.width.round() as u16,
                           height: layout.size.height.round() as u16,
                       });
                   }
               }
           }

           pub fn rect_for(&self, id: WidgetId) -> Option<Rect> {
               self.layout_cache.get(&id).copied()

               // (Taffy requires children detached before parent removal)
               // Remove from node_map and layout_cache
           }

           pub fn layout_cache(&self) -> &HashMap<WidgetId, Rect> {
       ```

    ```rust
    use std::collections::HashMap;
    use ratatui::layout::Rect;
    use crate::widget::WidgetId;

    pub struct MouseHitMap {
        // Sparse map: (col, row) -> WidgetId
        // Only stores cells that belong to a widget
        cells: HashMap<(u16, u16), WidgetId>,
    }

    impl MouseHitMap {
        /// Build hit map from layout cache.
        /// `widgets_dfs` is the DFS order of widget IDs from the screen root.
        /// Later entries in DFS order (deeper / later siblings) overwrite earlier ones,
        /// providing correct z-ordering for overlapping widgets (e.g., modals).
        pub fn build(
            widgets_dfs: &[WidgetId],
            layout_cache: &HashMap<WidgetId, Rect>,
        ) -> Self {
            let mut cells = HashMap::new();
            for &wid in widgets_dfs {
                if let Some(&rect) = layout_cache.get(&wid) {
                    for row in rect.y..rect.y.saturating_add(rect.height) {
                        for col in rect.x..rect.x.saturating_add(rect.width) {
                            cells.insert((col, row), wid);
                        }
                    }
                }
            }
            MouseHitMap { cells }
        }

        pub fn hit_test(&self, col: u16, row: u16) -> Option<WidgetId> {
            self.cells.get(&(col, row)).copied()
        }
    }
    ```

    Note on performance: For Phase 2 terminals (typical max ~300x80 = 24K cells), a HashMap is fine. For huge terminal sizes, a spatial index could be used later, but that's premature optimization.
