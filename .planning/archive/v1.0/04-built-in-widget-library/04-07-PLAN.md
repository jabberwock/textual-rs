    **tabs.rs (WIDGET-16):**

    Tabs struct (the tab bar):
    ```rust
    pub struct Tabs {
        pub tab_labels: Vec<String>,
        pub active: Reactive<usize>,
        own_id: Cell<Option<WidgetId>>,
    }
    ```

    Constructor: `Tabs::new(labels: Vec<String>) -> Self` with active=0.

    Messages:
    ```rust
    pub mod messages {
        pub struct TabChanged { pub index: usize, pub label: String }
        impl Message for TabChanged {}
    }
    ```

    `widget_type_name` "Tabs", `can_focus` true.

    Key bindings:
    - Left → "prev_tab" (show: false), Right → "next_tab" (show: false)

    `on_action`:
    - "prev_tab": if active > 0, decrement. Post TabChanged.
    - "next_tab": if active < tab_labels.len()-1, increment. Post TabChanged.

    `render`:
    - Draw tab bar: `" Label1 | Label2 | Label3 "` — active tab rendered with highlight (reverse video or underline). Separator is `" | "`.
    - Use `get_untracked()` for active index.

    `default_css`: `"Tabs { height: 1; dock: top; }"`

    TabbedContent struct (container holding Tabs + panes):
    ```rust
    pub struct TabbedContent {
        pub tabs: Tabs,
        pub panes: Vec<Box<dyn Widget>>,
    }
    ```

    Constructor: `TabbedContent::new(labels: Vec<String>, panes: Vec<Box<dyn Widget>>) -> Self`.

    `widget_type_name` "TabbedContent", `can_focus` false.

    `compose`: return `vec![Box::new(self.tabs.clone()), self.panes[active].clone()]` — but since we can't clone Box<dyn Widget>, the approach must be different. Instead:
    - TabbedContent renders the tabs bar itself in the top row of its area, then renders only the active pane in the remaining area.
    - `render`: draw tab bar in first row (delegate to Tabs::render), then render panes[active] in the remaining area (area.y+1, area.height-1).
    - Since panes are owned, render the active one directly.

    Actually, the simplest approach: TabbedContent is NOT a compose-based widget. It directly renders the tab bar and the active pane in its render method. Tabs is embedded, not composed.

    `default_css`: `"TabbedContent { min-height: 3; }"`

    }

    Constructor: `Collapsible::new(title: &str, children: Vec<Box<dyn Widget>>) -> Self` with expanded=true.

    `widget_type_name` "Collapsible", `can_focus` true.

    Key bindings:
    - Enter → "toggle" (description: "Toggle", show: true)

    `on_action("toggle")`: flip expanded. Post Expanded or Collapsed.

    `render` (Pitfall 6 — do NOT rely on compose for dynamic children):
    - Always render title row: `"▶ {title}"` if collapsed, `"▼ {title}"` if expanded
    - If expanded, render children in the area below the title row (area.y+1, area.height-1)
    - If collapsed, only render the title row — children are not rendered
    - This is a render-time approach, not a compose-time approach (per Pitfall 6)
    - Use `get_untracked()` for expanded state

    `default_css`: `"Collapsible { min-height: 1; }"`

    Add `pub mod tabs;` and `pub mod collapsible;` to widget/mod.rs.

    **markdown.rs (WIDGET-15, per D-06):**

    Struct:
    ```rust
    pub struct Markdown {
        pub content: String,
        rendered_lines: RefCell<Vec<RenderedLine>>,  // pre-computed on creation
    }

    struct RenderedLine {
        text: String,
        style: ratatui::style::Style,
        indent: u16,
    }
    ```

    Constructor: `Markdown::new(content: &str) -> Self` — parses content immediately, stores rendered lines.

    Method `fn parse_markdown(content: &str) -> Vec<RenderedLine>`:
    Use `pulldown_cmark::Parser::new(content)` to iterate events. Map events to RenderedLine entries:

    - `Event::Start(Tag::Heading { level, .. })` → push line with bold + bright style. Level maps to size: H1 gets underline, H2-H6 get bold only.
    - `Event::Text(text)` → append text to current line
    - `Event::Code(code)` → append `` `code` `` with dim/italic style
    - `Event::Start(Tag::CodeBlock(_))` → start a code block region, render with bordered block style (indent=2)
    - `Event::Start(Tag::Emphasis)` → set italic in style stack
    - `Event::Start(Tag::Strong)` → set bold in style stack
    - `Event::Start(Tag::List(None))` → bullet list, each Item gets `"  * "` prefix
    - `Event::Start(Tag::List(Some(start)))` → numbered list, each Item gets `"  {n}. "` prefix
    - `Event::Start(Tag::Link { dest_url, .. })` → after link text, append ` [{dest_url}]` in dim style
    - `Event::Rule` → horizontal rule `"────────"` repeated to width
    - `Event::SoftBreak` / `Event::HardBreak` → new line

    Not rendered (per D-06): images, tables, HTML.

    `widget_type_name` "Markdown", `can_focus` false.

    `render`:
    - Iterate rendered_lines from offset (if scrollable, but Markdown is non-scrollable in v1 — content just truncates to area height)
    - For each RenderedLine, write text with style at (area.x + indent, row_y)
    - Truncate lines to area.width

    `default_css`: `"Markdown { min-height: 3; }"`

    Add `pub mod markdown;` to widget/mod.rs. Add `pub use widget::markdown::Markdown;` to lib.rs.

    **Tests in widget_tests.rs** (append):
    - `snapshot_markdown_headings`: Markdown with "# H1\n## H2\n### H3", snapshot

    - `snapshot_markdown_code_block`: Markdown with "```\ncode here\n```", snapshot shows bordered block

    - `snapshot_markdown_link`: Markdown with "[click here](https://example.com)", snapshot shows "click here [https://example.com]"
    - `snapshot_markdown_mixed`: Larger markdown document with mixed elements
