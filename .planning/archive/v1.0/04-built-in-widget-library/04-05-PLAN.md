    **list_view.rs (WIDGET-09, per D-03):**

    Struct:
    ```rust
    pub struct ListView {
        pub items: Vec<String>,
        pub selected: Reactive<usize>,
        pub scroll_offset: Reactive<usize>,
        own_id: Cell<Option<WidgetId>>,
    }
    ```

    Constructor: `ListView::new(items: Vec<String>) -> Self` with selected=0, scroll_offset=0.

    Messages:
    ```rust
    pub mod messages {
        pub struct Selected { pub index: usize, pub value: String }
        impl Message for Selected {}
        pub struct Highlighted { pub index: usize }
        impl Message for Highlighted {}
    }
    ```

    `widget_type_name` "ListView", `can_focus` true.

    Key bindings:
    - Up → "cursor_up" (show: false), Down → "cursor_down" (show: false)
    - Enter → "select" (description: "Select", show: true)
    - Home → "cursor_home" (show: false), End → "cursor_end" (show: false)

    `on_action`:
    - "cursor_up": decrement selected (clamp 0). If selected < scroll_offset, set scroll_offset = selected. Post Highlighted.
    - "cursor_down": increment selected (clamp items.len()-1). If selected >= scroll_offset + viewport_height, scroll_offset = selected - viewport_height + 1. Post Highlighted.
    - "select": post Selected { index: selected, value: items[selected].clone() }
    - "cursor_home": set selected=0, scroll_offset=0. Post Highlighted.
    - "cursor_end": set selected=items.len()-1, adjust scroll_offset. Post Highlighted.

    NOTE: viewport_height must be stored or computed. Use a `viewport_height: Cell<u16>` field set in render() from area.height.

    `render`:
    - Read selected and scroll_offset with `get_untracked()`
    - Store `area.height` into `viewport_height` Cell
    - Render items from scroll_offset to min(scroll_offset + area.height, items.len())
    - Highlight selected item (reverse video style)
    - Right column (area.x + area.width - 1): draw scrollbar. Calculate thumb position: `(scroll_offset as f32 / max_offset as f32 * (area.height - 1) as f32) as u16`. Draw `"█"` at thumb position, `"│"` elsewhere.

    `default_css`: `"ListView { border: tall; min-height: 3; }"`

    **scroll_view.rs (WIDGET-19, per D-03):**

    Struct:
    ```rust
    pub struct ScrollView {
        pub scroll_offset_x: Reactive<usize>,
        pub scroll_offset_y: Reactive<usize>,
        pub children: Vec<Box<dyn Widget>>,
        viewport_width: Cell<u16>,
        viewport_height: Cell<u16>,
    }
    ```

    Constructor: `ScrollView::new(children: Vec<Box<dyn Widget>>) -> Self`.

    `widget_type_name` "ScrollView", `can_focus` true.

    `compose` returns self.children.

    Key bindings:
    - Up → "scroll_up", Down → "scroll_down"
    - Left → "scroll_left", Right → "scroll_right"
    - PageUp → "page_up", PageDown → "page_down"

    `on_action`:
    - "scroll_up": decrement scroll_offset_y (clamp 0)
    - "scroll_down": increment scroll_offset_y (clamp to max_scroll_y = content_height - viewport_height)
    - "scroll_left": decrement scroll_offset_x (clamp 0)
    - "scroll_right": increment scroll_offset_x (clamp to max_scroll_x)
    - "page_up": scroll_offset_y -= viewport_height (clamp 0)
    - "page_down": scroll_offset_y += viewport_height (clamp max)

    `render`:
    - Store viewport dimensions from area
    - The tricky part: ScrollView needs to render children into a virtual buffer, then blit the visible portion into the actual buffer.
    - Approach: Create a larger `Buffer` (content_width x content_height), render children into it, then copy the visible rect (scroll_offset_x..scroll_offset_x+viewport_width, scroll_offset_y..scroll_offset_y+viewport_height) into the actual buf.
    - For v1, content_height is estimated as sum of children's min heights or a configured `content_height` parameter.
    - Vertical scrollbar: right column, same algorithm as ListView
    - Horizontal scrollbar: bottom row if scroll_offset_x > 0 or content_width > viewport_width
    - Use `get_untracked()` for scroll offsets

    NOTE: The virtual buffer approach may be expensive. For v1, keep it simple — if performance is an issue, it can be optimized in v2 with lazy rendering. The current TestApp infrastructure works with small buffers.

    `default_css`: `"ScrollView { overflow: auto; }"`

    Add `pub mod scroll_view;` to widget/mod.rs. Add `pub use widget::scroll_view::ScrollView;` to lib.rs.

    - `snapshot_scroll_view_with_content`: ScrollView containing Labels that exceed viewport, snapshot at 20x5
    - `scroll_view_page_down`: Press PageDown, verify scroll_offset_y jumps by viewport_height
