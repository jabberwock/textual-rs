    `widget_type_name` returns "TextArea". `can_focus` returns true.

    Key bindings:
    - Up → "cursor_up", Down → "cursor_down"
    - Left → "cursor_left", Right → "cursor_right"
    - Home → "cursor_home", End → "cursor_end"
    - Ctrl+Left → "word_left", Ctrl+Right → "word_right"
    - Backspace → "delete_back", Delete → "delete_forward"
    - Enter → "newline"
    - Ctrl+C → "copy" (show: false), Ctrl+V → "paste" (show: false)
    - Ctrl+A → "select_all" (show: false)

    `on_event` for character insertion: match `KeyEvent { code: KeyCode::Char(c), modifiers: NONE | SHIFT }` — insert char at (cursor_row, cursor_col) in lines, advance cursor, post Changed.

    `on_action` handlers:
    - "cursor_up": decrement cursor_row (clamp to 0), adjust cursor_col to min(cursor_col, line_len)
    - "cursor_down": increment cursor_row (clamp to lines.len()-1), adjust cursor_col
    - "cursor_left": if cursor_col > 0 decrement, else if cursor_row > 0 go to end of previous line
    - "cursor_right": if cursor_col < line_len increment, else if cursor_row < last_line go to start of next line
    - "cursor_home": set cursor_col to 0
    - "cursor_end": set cursor_col to current line length
    - "word_left"/"word_right": scan for word boundary (same logic as Input)
    - "delete_back": if cursor_col > 0, remove char before cursor; if cursor_col == 0 and cursor_row > 0, join with previous line
    - "delete_forward": if cursor_col < line_len, remove char at cursor; if at end of line, join with next line
    - "newline": split current line at cursor_col, insert new line after, move cursor to start of new line. Post Changed.
    - "copy": read selection (if any) or current line, write to `arboard::Clipboard::new()?.set_text(text)`. Wrap in try-catch — clipboard may fail silently.
    - "paste": read from `arboard::Clipboard::new()?.get_text()`. If multiline, split and insert lines. Post Changed. Wrap in try-catch.
    - "select_all": selection tracking is a stretch — for v1, this is a no-op (selection via Shift+arrow is deferred to a later refinement if time permits)

    NOTE on selection (D-04): Selection (Shift+arrow) adds significant complexity. For v1, implement cursor movement and basic editing first. If selection is too complex for the context budget, document it as a gap and move on.

    `render`:
    - Read lines with `get_untracked()`
    - If show_line_numbers, reserve left margin (width = digit count of total lines + 1 space)
    - Render visible lines from scroll_offset, limited by area.height
    - Show cursor as reverse-video character at (cursor_row - scroll_offset, cursor_col + margin_offset)
    - Auto-scroll: if cursor_row < scroll_offset, scroll up; if cursor_row >= scroll_offset + area.height, scroll down

    `default_css`: `"TextArea { border: tall; min-height: 5; }"`

    Add `pub mod text_area;` to widget/mod.rs. Add `pub use widget::text_area::TextArea;` to lib.rs.
