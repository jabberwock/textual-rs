    **layout.rs (WIDGET-18):**
    - `pub struct Vertical { pub children: Vec<Box<dyn Widget>> }`
    - `pub struct Horizontal { pub children: Vec<Box<dyn Widget>> }`
    - Both: `can_focus` false, `compose` returns children
    - Vertical: `widget_type_name` "Vertical", `default_css`: `"Vertical { layout: vertical; width: 1fr; height: 1fr; }"`
    - Horizontal: `widget_type_name` "Horizontal", `default_css`: `"Horizontal { layout: horizontal; width: 1fr; height: 1fr; }"`
    - `render` is a no-op for containers — children render themselves in composed positions

    **header.rs (WIDGET-20):**
    - `pub struct Header { pub title: Reactive<String>, pub subtitle: Reactive<String> }`
    - Constructor: `Header::new(title: &str) -> Self` with empty subtitle
    - `widget_type_name` "Header", `can_focus` false
    - `render`: center `"{title} -- {subtitle}"` (or just title if subtitle empty) using `get_untracked()`
    - `default_css`: `"Header { dock: top; height: 1; background: $primary; color: $text; }"`

    **footer.rs (WIDGET-21):**
    - `pub struct Footer;` — no state, reads from context dynamically
    - `widget_type_name` "Footer", `can_focus` false
    - `render`: read `ctx.focused_widget`, if Some(id), read `ctx.arena[id].key_bindings()`, filter by `show == true`, render each as `" KEY Description "` separated by `"  "`. Use `get_untracked()` if any reactive reads involved. If no focused widget or no bindings, render empty bar.
    - `default_css`: `"Footer { dock: bottom; height: 1; background: $primary; color: $text; }"`
    - NOTE (Pitfall 4): arena read is safe because render holds only `&AppContext` (shared borrow)

    **placeholder.rs (WIDGET-22):**
    - `pub struct Placeholder { pub label: Option<String> }`
    - Constructor: `Placeholder::new() -> Self` with label=None
    - `widget_type_name` "Placeholder", `can_focus` false
    - `render`: center `"{label or 'Placeholder'}\n{width}x{height}"` showing the actual area dimensions
    - `default_css`: `"Placeholder { border: rounded; min-height: 3; min-width: 10; }"`

    Add module declarations to widget/mod.rs:

    **progress_bar.rs (WIDGET-12):**
    - Struct: `pub struct ProgressBar { pub progress: Reactive<Option<f64>> }` — None = indeterminate (per D-08)
    - Constructor: `ProgressBar::new(progress: f64) -> Self` wrapping in Some
    - Constructor: `ProgressBar::indeterminate() -> Self` with None
    - `widget_type_name` "ProgressBar", `can_focus` false
    - `render` (determinate): progress 0.0-1.0 mapped to fill width. Fill with `"█"` for filled portion, `"░"` for remaining. Use `get_untracked()`.
    - `render` (indeterminate): use a `tick: Cell<u8>` field that increments on each render (mod total_width). Draw a 3-char `"███"` block at the tick position (bouncing). This is a simple approach — the bounce reverses direction at edges.
    - `default_css`: `"ProgressBar { height: 1; width: 1fr; }"`

    **sparkline.rs (WIDGET-13):**
    - Struct: `pub struct Sparkline { pub data: Reactive<Vec<f64>> }` (per D-08 style)
    - Constructor: `Sparkline::new(data: Vec<f64>) -> Self`
    - `widget_type_name` "Sparkline", `can_focus` false
    - `render`: normalize data to 0-7 range (min=0, max=data_max or 1.0 if all zeros). For each data point, index into block chars `"▁▂▃▄▅▆▇█"` (8 levels). One char per data point, clipped to widget width. Use `get_untracked()`.
    - `default_css`: `"Sparkline { height: 1; }"`

    Add `pub mod progress_bar;` and `pub mod sparkline;` to widget/mod.rs.
    Add to lib.rs:
    ```rust
    pub use widget::progress_bar::ProgressBar;
    pub use widget::sparkline::Sparkline;
    ```
