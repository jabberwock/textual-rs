    **data_table.rs (WIDGET-10):**

    Supporting types:
    ```rust
    pub struct ColumnDef {
        pub label: String,
        pub width: Option<u16>,  // None = auto-size to content
    }
    ```

    Struct:
    ```rust
    pub struct DataTable {
        pub columns: Vec<ColumnDef>,
        pub rows: Vec<Vec<String>>,           // row-major storage
        pub cursor_row: Reactive<usize>,
        pub cursor_col: Reactive<usize>,
        pub scroll_offset_row: Reactive<usize>,
        pub scroll_offset_col: Reactive<usize>,
        sort_column: Cell<Option<usize>>,
        sort_ascending: Cell<bool>,
        column_widths: RefCell<Vec<u16>>,      // pre-computed
        viewport_rows: Cell<u16>,
        own_id: Cell<Option<WidgetId>>,
    }
    ```

    Constructor: `DataTable::new(columns: Vec<ColumnDef>) -> Self` with empty rows.
    Method: `pub fn add_row(&mut self, row: Vec<String>)` — append row, recompute column widths.
    Method: `pub fn sort_by_column(&self, col: usize)` — sort rows Vec by column index. Toggle ascending/descending if same column. Use `unicode_width::UnicodeWidthStr::width()` for column width calculation (Pitfall 7).

    Column width computation (run on add_row and initial setup):
    For each column, width = max(header.width(), max(row[col].width() for each row)). If ColumnDef.width is Some, use that instead.

    Messages:
    ```rust
    pub mod messages {
        pub struct RowSelected { pub row: usize }
        impl Message for RowSelected {}
        pub struct SortChanged { pub column: usize, pub ascending: bool }
        impl Message for SortChanged {}
    }
    ```

    `widget_type_name` "DataTable", `can_focus` true.

    Key bindings:
    - Up → "cursor_up", Down → "cursor_down"
    - Left → "cursor_left", Right → "cursor_right"
    - Enter → "select_row" (description: "Select", show: true)
    - Home → "cursor_home", End → "cursor_end"
    - `s` key → "sort_column" (description: "Sort", show: true) — sorts by current cursor_col

    `on_action`:
    - "cursor_up": decrement cursor_row, adjust scroll_offset_row if needed
    - "cursor_down": increment cursor_row, adjust scroll_offset_row
    - "cursor_left": decrement cursor_col, adjust scroll_offset_col
    - "cursor_right": increment cursor_col, adjust scroll_offset_col
    - "select_row": post RowSelected { row: cursor_row }
    - "sort_column": call sort_by_column(cursor_col.get_untracked()), post SortChanged
    - "cursor_home": cursor_row=0, scroll_offset_row=0
    - "cursor_end": cursor_row=last_row, adjust scroll

    `render`:
    - Header row: render column labels with separator `" | "` between columns. Sorted column gets `" ▲"` or `" ▼"` indicator.
    - Separator row: `"─"` repeated for each column width, `"┼"` at column boundaries
    - Data rows: render from scroll_offset_row, limited by viewport_rows. Each cell padded to column_width. Cursor cell highlighted with reverse video.
    - Vertical scrollbar on right column (same as ListView)
    - All reactive reads via `get_untracked()`

    `default_css`: `"DataTable { border: tall; min-height: 5; }"`

    Add `pub mod data_table;` to widget/mod.rs. Add `pub use widget::data_table::{DataTable, ColumnDef};` to lib.rs.

    - `data_table_cursor_navigation`: Create table, press Down/Right, verify cursor position
    - `data_table_select_row`: Navigate to row 2, press Enter, verify RowSelected { row: 2 }
    - `data_table_sort_column`: Press `s`, verify rows are sorted by current column, SortChanged emitted
    - `data_table_scroll_on_overflow`: Add 20 rows to 5-row viewport, navigate past viewport, verify scroll_offset_row > 0

    **tree_view.rs (WIDGET-11) — named tree_view to avoid conflict with widget/tree.rs (Pitfall 5):**

    Supporting types:
    ```rust
    pub struct TreeNode {
        pub label: String,
        pub data: Option<String>,
        pub children: Vec<TreeNode>,
        pub expanded: bool,
    }

    impl TreeNode {
        pub fn new(label: &str) -> Self { ... }
        pub fn with_children(label: &str, children: Vec<TreeNode>) -> Self { ... }
    }

    /// A flattened view entry for rendering
    struct FlatEntry {
        label: String,
        depth: usize,
        is_last_sibling: bool,
        has_children: bool,
        expanded: bool,
        path: Vec<usize>,   // path of indices from root to this node
    }
    ```

    Tree struct:
    ```rust
    pub struct Tree {
        pub root: TreeNode,
        pub cursor: Reactive<usize>,           // index into flattened list
        pub scroll_offset: Reactive<usize>,    // per D-03
        flat_entries: RefCell<Vec<FlatEntry>>,  // rebuilt on expand/collapse
        viewport_height: Cell<u16>,
        own_id: Cell<Option<WidgetId>>,
    }
    ```

    Constructor: `Tree::new(root: TreeNode) -> Self`.

    Method `fn rebuild_flat_entries(&self)`: walk tree in pre-order, skip collapsed subtrees, populate flat_entries.

    Messages:
    ```rust
    pub mod messages {
        pub struct NodeSelected { pub path: Vec<usize> }
        impl Message for NodeSelected {}
        pub struct NodeExpanded { pub path: Vec<usize> }
        impl Message for NodeExpanded {}
        pub struct NodeCollapsed { pub path: Vec<usize> }
        impl Message for NodeCollapsed {}
    }
    ```

    `widget_type_name` "Tree", `can_focus` true.

    Key bindings:
    - Up → "cursor_up", Down → "cursor_down"
    - Enter → "select" (description: "Select", show: true)
    - Space → "toggle" (description: "Expand/Collapse", show: true)
    - Right → "expand" (show: false) — expand if collapsed, else move to first child
    - Left → "collapse" (show: false) — collapse if expanded, else move to parent

    `on_action`:
    - "cursor_up"/"cursor_down": same pattern as ListView with scroll adjustment
    - "toggle": if current node has_children, flip expanded on the TreeNode at path. Call rebuild_flat_entries(). Post NodeExpanded or NodeCollapsed.
    - "expand": if collapsed and has children, expand. If already expanded, move cursor to first child.
    - "collapse": if expanded, collapse. If leaf or already collapsed, move to parent.
    - "select": post NodeSelected { path } for current flat entry

    Accessing TreeNode by path: walk `root.children[path[0]].children[path[1]]...` — helper method `fn node_at_path_mut(&mut self, path: &[usize]) -> &mut TreeNode`.

    NOTE: Since Widget methods take `&self`, the root TreeNode must be wrapped in `RefCell<TreeNode>` for mutation during expand/collapse. Update struct: `pub root: RefCell<TreeNode>`.

    `render`:
    - Rebuild flat entries (if dirty or first render)
    - Read cursor and scroll_offset with `get_untracked()`
    - For each visible flat entry, draw guide chars based on depth and is_last_sibling:
      - Prefix: for each depth level, if parent is_last_sibling `"    "` else `"│   "`
      - Connector: if is_last_sibling `"└── "` else `"├── "`
      - Expand indicator: if has_children, expanded `"▼ "` else `"▶ "`
      - Label text
    - Highlight cursor row with reverse video
    - Scrollbar on right column (same as ListView)

    `default_css`: `"Tree { border: tall; min-height: 5; }"`

    Add `pub mod tree_view;` to widget/mod.rs (NOT `pub mod tree;` — Pitfall 5).
    Add `pub use widget::tree_view::{Tree, TreeNode};` to lib.rs.

    **Tests in widget_tests.rs** (append):
    - `snapshot_tree_expanded`: Tree with 3 levels, all expanded. Snapshot shows guide chars.
    - `snapshot_tree_collapsed`: Same tree, root collapsed. Shows only root.
    - `tree_navigate_and_expand`: Navigate Down, press Space to expand, verify flat entries increased
    - `tree_collapse_node`: Expand a node, press Space again, verify flat entries decreased
    - `tree_select_emits_message`: Navigate to a leaf, press Enter, verify NodeSelected with correct path
    - `tree_scroll_on_overflow`: Large tree exceeding viewport, navigate past visible area, verify scroll_offset > 0
