    **Step 1: Create testing/mod.rs (per D-11, D-12)**

    The TestApp struct wraps an App with a TestBackend terminal. It does NOT run the full async event loop — instead it processes events synchronously via a `process_event` method, allowing tests to control timing precisely.

    ```rust
    // crates/textual-rs/src/testing/mod.rs
    pub mod pilot;
    pub mod assertions;

    use ratatui::backend::TestBackend;
    use ratatui::Terminal;
    use reactive_graph::owner::Owner;
    use any_spawner::Executor;

    use crate::app::App;
    use crate::event::AppEvent;
    use crate::widget::context::AppContext;
    use crate::widget::Widget;

    pub use pilot::Pilot;

    /// Headless test harness. Creates an App with TestBackend for automated testing.
    pub struct TestApp {
        app: App,
        terminal: Terminal<TestBackend>,
        tx: flume::Sender<AppEvent>,
        rx: flume::Receiver<AppEvent>,
        _owner: Owner,
    }

    impl TestApp {
        /// Create a new TestApp with the given root screen factory.
        /// Initializes reactive runtime, mounts the screen, and renders initial frame.
        ///
        /// # Example
        /// ```no_run
        /// let test_app = TestApp::new(80, 24, || Box::new(MyScreen::new()));
        /// ```
        pub fn new(
            cols: u16,
            rows: u16,
            factory: impl FnOnce() -> Box<dyn Widget> + 'static,
        ) -> Self {
            // Init reactive runtime (safe to call multiple times)
            let _ = Executor::init_tokio();
            let owner = Owner::new();

            let mut app = App::new(factory);

            let (tx, rx) = flume::unbounded::<AppEvent>();
            // Store tx on AppContext so widgets can post events
            // Need to access app internals — add a method to App for test setup
            app.set_event_tx(tx.clone());

            let backend = TestBackend::new(cols, rows);
            let mut terminal = Terminal::new(backend).unwrap();

            // Mount screen and initial render
            app.mount_root_screen();
            app.render_to_terminal(&mut terminal).unwrap();

            TestApp { app, terminal, tx, rx, _owner: owner }
        }

        /// Get a Pilot for sending simulated input.
        pub fn pilot(&mut self) -> Pilot<'_> {
            Pilot::new(self)
        }

        /// Access the AppContext for assertions.
        pub fn ctx(&self) -> &AppContext {
            self.app.ctx()
        }

        /// Access the rendered buffer for snapshot/assertion testing.
        pub fn buffer(&self) -> &ratatui::buffer::Buffer {
            self.terminal.backend().buffer()
        }

        /// Access the terminal backend (implements Display for insta snapshots).
        pub fn backend(&self) -> &TestBackend {
            self.terminal.backend()
        }

        /// Process a single event and re-render.
        pub fn process_event(&mut self, event: AppEvent) {
            // Process the event through dispatch logic
            // This mirrors the App event loop but synchronously
            match &event {
                AppEvent::Key(k) if k.kind == crossterm::event::KeyEventKind::Press => {
                    // Key binding check + dispatch to focused widget + bubble
                    self.app.handle_key_event(*k);
                }
                AppEvent::Mouse(m) => {
                    self.app.handle_mouse_event(*m);
                }
                AppEvent::RenderRequest | AppEvent::Resize(_, _) => {
                    // Just re-render
                }
                _ => {}
            }
            // Drain message queue
            self.app.drain_message_queue();
            // Re-render
            self.app.render_to_terminal(&mut self.terminal).unwrap();
        }
    }
    ```

    **Important:** This requires refactoring some App methods to be callable from TestApp. Add these helper methods to App:
    - `pub fn set_event_tx(&mut self, tx: flume::Sender<AppEvent>)` — sets `self.ctx.event_tx`
    - `pub fn mount_root_screen(&mut self)` — takes the factory, calls push_screen
    - `pub fn render_to_terminal<B: Backend>(&mut self, terminal: &mut Terminal<B>) -> Result<()>` — the full_render_pass logic, made public for TestApp
    - `pub fn handle_key_event(&self, key: KeyEvent)` — the key dispatch logic (check bindings, dispatch to focused, handle Tab)
    - `pub fn handle_mouse_event(&self, mouse: MouseEvent)` — mouse hit test + dispatch

    These are extracted from run_async's event loop match arms into reusable methods. The run_async event loop then calls these same methods. This refactoring keeps the code DRY.

    Note: `handle_key_event` takes `&self` (not `&mut self`) because on_event and key_bindings both take `&self`. Focus changes (Tab) need `&mut self` though — handle_key_event returns a flag indicating if Tab was pressed, and the caller does the focus advance with `&mut self.ctx`.

    Actually, revise: make `handle_key_event` and `handle_mouse_event` take `&mut self` since they may need to mutate AppContext (focus changes). The borrow is fine because these are called from the event loop where we already have `&mut self`.

    ```rust
    // crates/textual-rs/src/testing/pilot.rs
    use crossterm::event::{KeyCode, KeyEvent, KeyEventKind, KeyModifiers, MouseButton, MouseEvent, MouseEventKind};
    use crate::event::AppEvent;
    use super::TestApp;

    pub struct Pilot<'a> {
        test_app: &'a mut TestApp,
    }

    impl<'a> Pilot<'a> {
        pub(super) fn new(test_app: &'a mut TestApp) -> Self {
            Pilot { test_app }
        }

        /// Simulate pressing a key.
        pub async fn press(&mut self, key: KeyCode) -> &mut Self {
            self.press_with_modifiers(key, KeyModifiers::NONE).await
        }

        /// Simulate pressing a key with modifiers.
        pub async fn press_with_modifiers(&mut self, key: KeyCode, modifiers: KeyModifiers) -> &mut Self {
            let event = AppEvent::Key(KeyEvent::new(key, modifiers));
            self.test_app.process_event(event);
            self.settle().await;
            self
        }

        /// Type a string by sending each character as a key press.
        pub async fn type_text(&mut self, text: &str) -> &mut Self {
            for ch in text.chars() {
                let event = AppEvent::Key(KeyEvent::new(KeyCode::Char(ch), KeyModifiers::NONE));
                self.test_app.process_event(event);
            }
            self.settle().await;
            self
        }

        /// Simulate a mouse click at (col, row).
        pub async fn click(&mut self, col: u16, row: u16) -> &mut Self {
            let down = AppEvent::Mouse(MouseEvent {
                kind: MouseEventKind::Down(MouseButton::Left),
                column: col,
                row,
                modifiers: KeyModifiers::NONE,
            });
            self.test_app.process_event(down);
            self.settle().await;
            self
        }

        /// Drain all pending events and reactive effects until quiescent.
        /// Uses a bounded loop (max 100 iterations) to prevent infinite loops.
        pub async fn settle(&mut self) {
            for _ in 0..100 {
                // Yield to let reactive effects fire
                tokio::task::yield_now().await;

                // Process any events generated by effects (e.g., RenderRequest)
                while let Ok(event) = self.test_app.rx.try_recv() {
                    self.test_app.process_event(event);
                }

                // Drain message queue
                self.test_app.app.drain_message_queue();

                // If nothing more to process, we're quiescent
                if self.test_app.rx.is_empty()
                    && self.test_app.app.ctx().message_queue.borrow().is_empty()
                {
                    break;
                }
            }
            // Final render to capture settled state
            self.test_app.app.render_to_terminal(&mut self.test_app.terminal).unwrap();
        }
    }
    ```

    Note: `KeyEvent::new(key, modifiers)` creates an event with `kind: KeyEventKind::Press` by default in crossterm 0.29.

    // crates/textual-rs/src/testing/assertions.rs
    use ratatui::buffer::Buffer;

    /// Assert that specific rows of the buffer match expected strings.
    /// Each string corresponds to one row of the buffer, starting from row 0.
    /// Trailing spaces in the buffer are trimmed for comparison.
    ///
    /// # Panics
    /// Panics with a diff if any row does not match.
    pub fn assert_buffer_lines(buffer: &Buffer, expected: &[&str]) {
        let area = buffer.area;
        for (row_idx, expected_line) in expected.iter().enumerate() {
            assert!(
                (row_idx as u16) < area.height,
                "Expected line {} but buffer only has {} rows",
                row_idx, area.height
            );
            let mut actual = String::new();
            for col in 0..area.width {
                let cell = &buffer[(col, row_idx as u16)];
                actual.push_str(cell.symbol());
            }
            let actual_trimmed = actual.trim_end();
            assert_eq!(
                actual_trimmed, *expected_line,
                "Row {} mismatch:\n  expected: {:?}\n  actual:   {:?}",
                row_idx, expected_line, actual_trimmed
            );
        }
    }

    /// Assert that a specific cell at (col, row) contains the expected character.
    pub fn assert_cell(buffer: &Buffer, col: u16, row: u16, expected: &str) {
        let cell = &buffer[(col, row)];
        assert_eq!(
            cell.symbol(), expected,
            "Cell ({}, {}) mismatch: expected {:?}, got {:?}",
            col, row, expected, cell.symbol()
        );
    }
